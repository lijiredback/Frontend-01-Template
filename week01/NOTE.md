## 2. 什么是好的前端

### 职业规划

+ 成就最重要（你做了什么？做出了什么？如何体现？）
+ 成就的三个方向：
  - 业务型成就
  - 工程型成就（最容易）
    - 目标：质量、效率
    - 方案与实施（规章制度、库、工具、系统）
    - 结果（线上监控）
  - 技术难题（公认的技术难题）

### 数据驱动的思考方式

循环：
+ 目标：分析业务目标，定数据指标
+ 现状：采集数据，建立数据展示系统
+ 方案：设计技术方案，预估数据
+ 实施：小规模实验优先，再大规模推广落地
+ 结果：统计最终效果汇报

### 业务敏感度

+ 技术数据
  - 加载数据
  - 错误数据
+ 业务数据
+ 预估数据

### 前端技能模型

+ 领域知识
+ 前端知识
+ 能力
  - 编程能力
  - 架构能力
  - 工程能力

### 技术架构

+ 库：有复用价值的代码
  - URL
  - AJAX
  - ENV
+ 组件：UI 上多次出现的元素
  - 轮播
  - Tab
+ 模块：经常被使用的业务代码
  - 登录

### 前端架构：解决大量页面需求带来的重复劳动（复用）
